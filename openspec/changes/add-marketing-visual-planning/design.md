## Context
本变更面向“产品为中心”的营销视觉策划：先生成三条差异化策略路线与海报级提示词，再基于选定路线输出 8 张图内容规划。系统以 `prompt.md` 的约束为准，不引入品牌信息输入与数据持久化。

## Goals / Non-Goals
- Goals:
  - 阶段一生成：产品分析 + 3 条策略路线 + 每条 3 个海报提示词
  - 阶段二生成：选定路线下的 8 张图内容规划（2 张主图 + 6 张长图）
  - 保证字数、数量、顺序与语言规则满足 `prompt.md`
  - 视觉一致性：阶段二继承阶段一路线的色彩与风格
- Non-Goals:
  - 品牌信息/网址输入与解析
  - 结果持久化与历史管理
  - 多产品并行策划与协作流程

## Decisions
- 采用独立页面 `/marketing-planner`，减少对现有电商图流程的耦合。
- 两个后端接口拆分阶段一与阶段二，便于校验与重试。
- 输入字段最小化：产品图必填；产品名称、卖点描述可选。
- 阶段一输出中包含颜色信息，并强制写入海报提示词；阶段二继承此颜色体系。
- 前端严格校验 JSON 结构与字数范围，错误即时提示。

## Alternatives considered
- 集成到现有电商图向导：路径短但耦合高、改动面大。
- 增加持久化与版本：可复用但超出最小闭环目标。

## Risks / Trade-offs
- LLM 输出不满足字数/结构：通过 schema 校验与错误提示降低风险。
- 颜色提取不稳定：以“产品图视觉分析 + 文本约束”优先满足一致性，后续可引入调色板提取。
- 两阶段依赖：阶段一失败会阻塞阶段二；提供重试与局部重生成。

## Migration Plan
无需迁移，新增能力独立上线。
