## Context
电商图生成是一个专业工具，需要用户完成多个步骤：选择产品 → 选择参考图 → 配置模板选项 → 生成图片。当前单页滚动式设计导致用户容易迷失，字体过小影响可读性。

### 目标用户
- 电商运营人员、设计人员
- 需要快速生成商品图的小商家
- 使用台式机或笔记本进行操作

### 约束条件
- 保持与现有后端 API 的兼容性
- 不改变核心功能逻辑，只优化呈现
- 需要兼容 1024px-2560px 屏幕宽度

## Goals / Non-Goals

### Goals
- 让用户一眼看清当前在哪一步、已完成什么、还需要做什么
- 通过增大字体和间距提升可读性
- 通过分组收缩减少模板选项区的视觉拥挤
- 支持响应式布局，在不同屏幕尺寸下都有良好体验

### Non-Goals
- 不改变产品选择、模板编辑、图片生成的业务逻辑
- 不重新设计后端 API
- 不添加新功能，只优化现有功能的呈现

## Decisions

### Decision 1: 采用向导式分步设计
**选择理由**：
- 将认知负担分散到多个步骤，每步只需关注一件事
- 顶部导航条让用户随时知道进度
- 可以在步骤间自由切换，不强制线性流程

**交互模式**：
```
步骤导航条：[ 1.选择产品 ] → [ 2.配置模板 ] → [ 3.预览提示词 ] → [ 4.生成图片 ]
                  ↑ 当前步骤高亮，已完成步骤带勾号
```

**备选方案**：
- 保持单页滚动：优点是无页面切换，缺点是信息拥挤问题难以解决
- 标签页切换：优点是紧凑，缺点是没有流程感

### Decision 2: 调整字体大小规范

| 元素 | 当前 | 优化后 |
|------|------|--------|
| 模板标签片 (chip) | 13px | 15px |
| 模板模块标题 | 14px | 16px |
| 面板标题 | 16px | 18px |
| 按钮文字 | 14px | 16px |
| 生成按钮 | 16px | 18px |
| 辅助说明文字 | 12px | 14px |
| 分组标签 (section title) | 12px | 14px |

### Decision 3: 模板选项分组收缩设计
```
┌─ 场景选择 ─────────────────────── [展开] ─┐
│  ○ 白色背景  ○ 自然光桌面  ○ 户外场景     │
│  ○ 纯色背景  ○ 生活场景   + 添加          │
└────────────────────────────────────────────┘

┌─ 拍摄角度 ─────────────────────── [收起] ─┐
│  已选: 正面、45度侧面 (2项)               │
└────────────────────────────────────────────┘
```

- 默认展开第一个未选择项目的分组
- 已选择内容在收起时显示摘要
- 点击标题展开/收起

### Decision 4: 响应式断点设计

| 断点 | 布局 |
|------|------|
| ≥1440px | 三栏：侧边栏(产品) + 主内容区 + 预览/结果面板 |
| 1024-1439px | 两栏：侧边栏 + 主内容区（向导步骤垂直堆叠） |
| <1024px | 单栏：顶部导航 + 内容区垂直堆叠 |

### Decision 5: 视觉层次优化
- 面板内边距从 20px 增加到 24-28px
- 标签片间距从 8px 增加到 12px
- 分组之间间距从 20px 增加到 28px
- 步骤指示器尺寸从 28px 增加到 36px

## Risks / Trade-offs

| 风险 | 缓解措施 |
|------|----------|
| 分步可能增加操作步骤感知 | 支持步骤导航快速跳转，保持流畅 |
| 大字体可能在小屏幕显得拥挤 | 响应式设计，小屏幕适当缩小 |
| 收缩面板可能隐藏重要选项 | 已选项在收起时显示摘要，点击即展开 |

## Open Questions

- 预览步骤是否可以与配置步骤合并？（保持分离更清晰）
