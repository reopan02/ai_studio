## Context
Portal 主页是用户登录后的第一入口，需要快速引导用户找到所需功能。当前设计过于技术化，缺乏视觉吸引力和清晰的信息层次。

### 目标用户
- 电商运营人员
- 内容创作者
- 产品经理

### 约束条件
- 保持与现有路由 URL 的兼容性（不改变 /video, /image 等路径）
- 保留全局 API 配置折叠面板
- 保留 Admin 权限判断逻辑

## Goals / Non-Goals

### Goals
- 让用户 3 秒内找到目标功能入口
- 通过视觉差异化帮助用户记忆各功能位置
- 提升品牌感和专业度

### Non-Goals
- 不改变各功能页面的具体实现
- 不修改后端 API
- 不添加新功能，只优化展示

### Decision 2: 渐变图标实现方式
使用 CSS linear-gradient 结合 SVG 的 mask 属性：
```css
.portal-icon-video {
  background: linear-gradient(135deg, #8B5CF6, #3B82F6);
  mask: url('icon.svg');
  -webkit-mask: url('icon.svg');
}
```

或使用 SVG defs 定义渐变：
```svg
<defs>
  <linearGradient id="grad-video" x1="0%" y1="0%" x2="100%" y2="100%">
    <stop offset="0%" style="stop-color:#8B5CF6"/>
    <stop offset="100%" style="stop-color:#3B82F6"/>
  </linearGradient>
</defs>
```

**选择**：使用 SVG defs 方式，更易于维护且兼容性更好。

### Decision 3: 响应式断点
| 屏幕宽度 | 每行卡片数 | 卡片宽度 |
|----------|-----------|---------|
| ≥1200px | 3 | ~360px |
| 768-1199px | 2 | ~360px |
| <768px | 1 | 100% |

### Decision 4: 分组标签样式
- 使用小型大写字母标签
- 左侧带装饰线条
- 颜色与该分组主色呼应

## Risks / Trade-offs

| 风险 | 缓解措施 |
|------|----------|
| SVG 渐变可能在某些浏览器渲染不一致 | 提供 fallback 纯色 |
| 分组后卡片位置变化可能影响老用户习惯 | 位置变化不大，且分组标签清晰 |
